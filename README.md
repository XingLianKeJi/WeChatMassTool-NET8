# WeChatMassTool - 微信群发消息工具

基于 .NET 8 WinForms 的微信群发消息桌面工具，通过 Win32 API 操控微信窗口，模拟键盘输入实现自动化群发。支持从微信本地数据库直接读取联系人和群聊数据。

## 功能特性

- **批量群发消息** — 支持逐条发送（每行一条）和整段发送模式
- **文件发送** — 支持批量添加文件并发送给多个联系人
- **联系人管理** — 手动输入、导入名单文件、从微信数据库导出联系人/群成员
- **定时发送** — 设定未来时间自动执行群发任务，支持多任务并行管理
- **断点续发** — 程序重启后自动从上次中断位置继续发送
- **发送间隔控制** — 可调节文本间隔和文件间隔，模拟人工操作节奏
- **OCR 智能搜索** — 利用 Windows OCR 精准匹配微信搜索结果
- **运行结果导出** — 将发送结果导出为 CSV 文件
- **自动更新** — 检测 GitHub Release 新版本并提示下载

## 使用指南

### 环境要求

- **操作系统**: Windows 10 1809+ / Windows 11
- **微信**: 需已登录 PC 版微信客户端
- **权限**: 读取微信数据库需要管理员权限运行

### 快速开始

#### 方式一：下载发布版

前往 [Releases](https://github.com/XingLianKeJi/WeChatMassTool-NET8/releases) 页面下载最新版本的发布包，解压后运行 `WeChatMassTool.exe`。

#### 方式二：自行编译

```bash
# 克隆项目
git clone https://github.com/XingLianKeJi/WeChatMassTool-NET8.git
cd WeChatMassTool-NET8

# 构建
dotnet build -c Release

# 运行
dotnet run

# 或发布为独立部署
dotnet publish -c Release -r win-x64
```

### 操作步骤

#### 1. 编写消息内容

在"消息"区域有两个输入框：

| 输入框         | 说明                     |
| ----------- | ---------------------- |
| **逐条发送**（左） | 每行视为一条独立消息，会依次发送给每个联系人 |
| **整段发送**（左） | 整个输入框内容作为一条消息发送        |

可以同时填写两个输入框，发送时会按顺序先发逐条消息，再发整段消息。

#### 2. 添加文件（可选）

点击"添加文件"按钮选择要发送的文件，支持多选。已添加的文件会显示在列表中，可点击"清空文件"移除全部。

#### 3. 添加发送对象

支持三种方式添加联系人：

- **手动输入** — 在"对象"文本框中输入好友昵称，一行一个
- **导入名单** — 从 `.txt` 文件导入，每行一个昵称
- **导出联系人** — 从微信本地数据库自动读取联系人列表（需管理员权限）
- **导出群成员** — 从微信群聊中提取成员昵称（需管理员权限）

#### 4. 配置发送参数

- **发送方式**: 选择 `Enter` 或 `Ctrl+Enter`（需与微信设置一致）
- **文本间隔**: 两条文本消息之间的等待时间（0.05s ~ 0.50s）
- **文件间隔**: 两个文件之间的等待时间（0.25s ~ 2.50s）

#### 5. 开始发送

点击"开始发送"按钮，程序会：

1. 激活并前置微信窗口
2. 依次搜索每个联系人
3. 粘贴消息内容并发送
4. 更新进度条和状态信息
5. 发送完成后可导出结果为 CSV

#### 定时发送

1. 在"定时"选择框中设置目标时间
2. 点击"定时发送"创建定时任务
3. 任务会显示在右侧面板中，实时显示倒计时
4. 到达指定时间后自动执行发送
5. 支持同时创建多个定时任务

定时任务特性：

- 任务数据持久化，重启程序后自动恢复
- 过期 5 分钟内的任务仍会执行
- 可点击任务卡片编辑或取消

### 运行结果导出

发送过程中可随时点击"导出结果"按钮，将发送记录保存为 CSV 文件，包含以下字段：

| 字段  | 说明          |
| --- | ----------- |
| 昵称  | 联系人昵称       |
| 文本  | 发送的文本内容     |
| 文件  | 发送的文件路径     |
| 状态  | 发送结果（成功/失败） |
| 备注  | 失败原因等信息     |

### 快捷键

| 快捷键      | 功能                  |
| -------- | ------------------- |
| `Ctrl+D` | 打开微信数据库诊断工具（排查解密问题） |

### 通知类型

操作反馈通过右上角 Toast 通知显示：

- **绿色** ✓ — 操作成功
- **红色** ✕ — 操作失败
- **蓝色** ℹ — 提示信息

## 工作原理

### 微信窗口操控

通过 Win32 API（`user32.dll`）查找微信主窗口（类名 `mmui::MainWindow`），模拟键盘操作实现自动化：

```
搜索联系人: Ctrl+F → Ctrl+V(粘贴昵称) → 方向键选择 → Enter
发送消息:   Ctrl+V(粘贴内容) → Enter / Ctrl+Enter
发送文件:   剪贴板设置文件路径(CF_HDROP) → Ctrl+V → Enter
```

### 微信数据库读取

通过分析微信进程内存获取 SQLCipher 加密密钥，解密本地 SQLite 数据库：

1. 定位微信数据目录（注册表 + 文件系统搜索）
2. 从微信进程内存中搜索 32 字节加密密钥（三种递进策略）
3. 使用密钥打开 SQLCipher 数据库（复制到临时目录避免锁定）
4. 读取联系人、群聊、群成员等数据

### 模拟人工节奏

内置 `HumanSimulator` 模拟人类操作节奏，包括高斯分布随机延迟、按键间随机停顿（8-25ms）等，降低被检测风险。

## 常见问题

### 提示"微信未启动"

启动程序前需先打开并登录 PC 版微信。程序依赖微信客户端进行消息发送。

### 导出联系人/群成员失败

- 确认以**管理员权限**运行 `WeChatMassTool.exe`
- 确认微信已登录且数据已同步
- 按 `Ctrl+D` 打开诊断工具排查数据库密钥搜索情况
- 如果自动检测数据目录失败，程序会提示手动选择目录

### 发送时找不到联系人

- 确认输入的昵称与微信中的完全一致
- 检查微信设置中的"发送快捷键"是否与程序设置匹配

### 程序已运行但找不到窗口

程序使用单实例锁，同一时间只能运行一个。检查任务栏或系统托盘是否已有实例。
